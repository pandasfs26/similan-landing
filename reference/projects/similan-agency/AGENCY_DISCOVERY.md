- **Influencer/Outreach Bot** ✅
- **Project Manager Bot** ✅
- **Security & Compliance Bot** ✅ (NEW)

**Total: 10 Bots**

**Security & Compliance Bot Responsibilities:**
- Monitor for credential leaks (API keys, passwords, tokens) in all bot outputs
- Scan generated content for malware, malicious payloads, suspicious links
- Review all .md files for security best practices (no hardcoded secrets, safe code patterns, etc.)
- Review all skill files against OpenClaw security guidelines + industry standards (OWASP, etc.)
- Flag suspicious activity, unauthorized access attempts, anomalies
- Regular security audits of bot outputs & team access
- Ensure compliance with Thai data protection laws (PDPA)

**Note:** All bots including Security bot to be set up immediately. Some content bots won't be fully active first 2 months, but Security runs 24/7 from day 1.

### C2. Workflow & Handoff
- **Cadence:** Build content calendar for each company, tailored by channel type (to be finalized)
- **Review cycle:** First 2 months: review everything before posting. After that: (model to be discussed)
  - **Security gate:** All content passes through Security Bot BEFORE Dan's review
  - Malware/malicious content automatically flagged and quarantined
  - Credential leaks automatically redacted
- **Reporting:** Yes to all — weekly roundup + monthly deep-dive + real-time dashboard
  - **Security reports included:** Weekly security digest (threats detected, compliance status)
- **Crisis/escalation:** Telegram alert for: customer reviews/comments on FB, Instagram, Google My Business
  - **+Security alerts:** Any suspicious activity, credential leaks, malware detection

### C3. Brand Asset Library & Security
**Action:** Build directory structure, Dan will populate per brand

**Needed:**
- Logo files + usage guidelines
- Color palettes / brand fonts
- Photo library (product, restaurant interiors, team)
- Video library
- Brand guidelines documents

**Monthly Feed:** Photographer provides fresh photos & videos monthly for bots to use in content creation

**Security Best Practices for .md & Skill Files:**
- **No hardcoded credentials** (API keys, passwords, tokens) — all secrets via secure environment variables
- **Safe code patterns** — no arbitrary exec(), use linting & static analysis
- **OpenClaw compliance** — follow skill spec, avoid unsafe permissions, no privilege escalation
- **OWASP standards** — input validation, output encoding, secure defaults
- **Thai PDPA** — no personal data exposure, proper consent handling
- **Security Bot will review** all new .md & skill files before they're used
