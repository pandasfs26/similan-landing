## BLUEPRINT 1: High-End Restaurant / Bar (Lucky 13 Template)

```
CONTEXT:
You are designing a website for a premium sandwich shop / cocktail bar with 
locations across Thailand. The brand is modern, energetic, locally-rooted, 
and attracts dine-in traffic and tourists.

CORE TONE & VALUES:
- Fresh, high-energy, approachable (not stuffy)
- Emphasis: local ingredients, crafted quality, community gathering
- Audience: 25-45 urban professionals, tourists, couples, small groups
- Primary goal: Drive dine-in foot traffic + location discovery

BRAND CONSTRAINTS:
- Primary color: #ED2124 (bold red)
- Secondary: #FFFFFF (clean white)
- Accent: #FFB3A7 (warm coral)
- Typography: Bebas Neue (headlines, bold), Montserrat Medium (body, readable)
- Photography style: Warm, lifestyle shots; food close-ups; staff/community
- Tone: Conversational, friendly, occasionally cheeky

PAGE STRUCTURE (required):
1. Hero: Full-width food/location image + headline CTA ("Reserve Your Table")
2. About: 2–3 sentences + founder story visual
3. Menu preview: 6–8 hero dishes with descriptions
4. Locations: Interactive map + hours + reservation links
5. Social proof: 3–5 testimonials + Instagram feed embed
6. CTA: "Book Now" + contact info

DESIGN REQUIREMENTS:
- Mobile-first responsive (hero image adapts, menu scrolls clean)
- Fast load (optimize food imagery)
- Accessibility: Large touch targets for reservation CTA
- SEO hooks: Location pages, menu schema markup
- Conversion: Reservation link (book.com or Resy integration)

STYLE PREFERENCES:
- Grid-based layout (clean, modern)
- Large typography (readable on mobile)
- White space (luxury feel, not cluttered)
- Minimal animation (food should be the star)

OUTPUT:
Generate 3 distinct layout directions:
A) Left-aligned hero + centered menu grid (editorial feel)
B) Full-width hero carousel + side-by-side locations (energetic)
C) Immersive food photography + minimal text overlays (luxury feel)

For each: Provide annotated screenshot with component callouts.
```

- Tone: Expert, approachable, detail-oriented

Generate 3 distinct layout directions:

C) Unified approach (one narrative that serves both audiences equally)

For each: Annotated screenshot showing B2B vs B2C pathways clearly.

## BLUEPRINT 4: Kitchen Safety Digital App (Tablet, Offline-First)

```
CONTEXT:
You are designing a production interface for a digital kitchen safety 
checklist app running on Android tablets (10-inch, landscape orientation). 
The app digitizes daily food safety compliance for hotel kitchens.

CORE TONE & VALUES:
- Clear, fast, zero ambiguity (lives in a busy kitchen)
- Emphasis: Speed (checks completed in <2 min), compliance proof, offline-ready
- Audience: Kitchen staff, shift supervisors, compliance auditors
- Primary goal: 100% checklist completion, audit trail, offline sync

CONTEXT CONSTRAINTS:
- Hardware: 10-inch Android tablet, landscape (1280×800 typical)
- Network: No WiFi guarantee; offline-first (data saved locally, sync when online)
- Users: Busy staff (may have wet hands, gloves); supervisors doing audits
- Language: Thai primary, English secondary
- Accessibility: High contrast, large buttons (cooking environment glare)

SCREEN STRUCTURE (required):
1. Home: List of due checkpoints + status (not started, in progress, done)
   - Large green button: "Start New Check"
   - Quick filter: By time of day or area
2. Checkpoint form: 
   - Single checkpoint per screen
   - Input types: Yes/No toggle, text note, photo evidence
   - "Pass" or "Issue" at bottom
3. Issue detail (if "Issue" selected):
   - Photo evidence capture
   - Brief description
   - Severity (minor, moderate, critical)
   - Escalation option: "Alert supervisor"
4. Supervisor view: Summary of shifts, issues, compliance % 
5. Sync status: Shows "X items waiting to sync" + retry button

DESIGN REQUIREMENTS:
- Landscape-optimized (landscape ≠ portrait rotated; design for long width)
- Offline-first: No spinning loaders; work immediately
- Touch-friendly: Buttons ≥48px, spacing for gloved hands
- High contrast: White text on dark backgrounds (kitchen lighting)
- Thai language support (character rendering)
- Minimal loading states (no auth screens; logged-in on first launch)

STYLE PREFERENCES:
- Flat, modern interface (no skeuomorphism; fast parsing)
- Color coding: Green = pass, yellow = issue, red = critical
- Typography: Sans-serif, bold labels (readability)
- Icons: Simple, culturally universal (no text-heavy labels)

OUTPUT:
Generate 3 distinct UI direction sets (each = 4 screens: home, form, issue, sync):
A) Horizontal card-based layout (left nav = status, right = form; maximize space)
B) Full-width form (home carousel → form → issue; swipe navigation)
C) Multi-pane layout (status list left, detail right; desktop-like control room feel)

For each: Annotated screenshots showing:
- Touch targets (button sizes, spacing)
- Offline indicator placement
- Sync mechanism visibility
- Language option toggle
```

## USAGE GUIDE

**How to use these prompts:**

1. **Copy the entire blueprint** (from opening ``` to closing ```)
2. **Substitute bracketed values:**
   - `[Name]` → Client name
   - `[X]` and `[Y]` → Specific outcomes
   - `[Client's primary]` → Brand color
3. **Paste into Stitch 2.0 text input**
4. **Hit "Generate"** → Wait for 3 layout directions
5. **Export:** Figma + HTML/React/Tailwind (Stitch will offer)

**Expected output per prompt:**
- 3 distinct UI/layout directions
- Annotated screenshots
- Component specifications
- Figma file + code (React/Tailwind)

## INTEGRATION WITH OPENCLAW BOTS

**Post-generation workflow:**

1. **intake-stitch bot:**
   - Collects client brief
   - Matches to blueprint (restaurant → Blueprint 1, etc.)
   - Generates Stitch prompt with client values substituted
   - Runs Stitch, exports 3 options

2. **prototype-review bot:**
   - Reviews Stitch outputs
   - Selects best direction
   - Exports Figma + code

3. **proposal-bot:**
   - Creates annotated screenshots
   - Drafts Loom script (5 min walkthrough)
   - Generates pricing email:
     - Design only: $2–3K
     - Design + build: $8–10K
     - Design + build + 3mo marketing: $15–20K

4. **dev-build-bot:**
   - Takes approved Figma + code from Stitch
   - Writes tickets for integration (forms, booking, analytics)
   - Drafts README + deployment notes

5. **content-bot:**
   - Uses BRAND_UI.md (extracted from Stitch output)
   - Ensures social posts match design direction
   - Cross-sells marketing packages
